## アーキテクチャパターン

### ドメイン駆動設計（DDD）レイヤードアーキテクチャ

**注**: 詳細なルールとコード例は `.claude/guides/ddd.md` を参照。

```
┌────────────────────────────────┐
│ presentation（プレゼンテーション層）│ ← CLI / API / UI
├────────────────────────────────┤
│ application（アプリケーション層）   │ ← ユースケースの調整役
├────────────────────────────────┤
│ domain（ドメイン層）★中心          │ ← エンティティ / 値オブジェクト /
│                                │    集約 / ドメインサービス / リポジトリIF
├────────────────────────────────┤
│ infrastructure（インフラ層）       │ ← DB / ファイル / 外部API（リポジトリ実装）
└────────────────────────────────┘

依存方向: presentation → application → domain ← infrastructure
```

#### domain（ドメイン層）
- **責務**: ビジネスルール、不変条件の実装（エンティティ・値オブジェクト・集約・ドメインサービス・リポジトリインターフェース）
- **許可される依存**: なし（標準ライブラリのみ）
- **禁止される操作**: 他層のimport、DB・API・フレームワークへの依存

#### application（アプリケーション層）
- **責務**: ユースケースの手順調整、トランザクション境界の管理
- **許可される依存**: domain層
- **禁止される操作**: ビジネスルールの実装（ドメイン層に置く）

#### infrastructure（インフラ層）
- **責務**: リポジトリインターフェースの実装、技術詳細（永続化・外部API）
- **許可される依存**: domain層（インターフェースの実装のため）
- **禁止される操作**: ビジネスルールの実装

#### presentation（プレゼンテーション層）
- **責務**: ユーザー入力の受付、形式バリデーション、結果の表示
- **許可される依存**: application層
- **禁止される操作**: domain層の直接操作、ビジネスルールの実装
